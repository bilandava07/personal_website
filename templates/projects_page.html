Multiplayer Hanabi Card Game
Originally developed as a university project with a team of four, I led backend development and game logic, while also helping less experienced team members with frontend, so that the project got a top grade in the end. Nevertheless, the initial design of the game was flawed and the game was only barely playable on local network. We stored game state on each client, only sending game moves to update it locally, which caused constant lag and crashes.

Over the holidays, I rewrote the MVC architecture and networking from scratch, centralizing the game state on the host's machine and enabling smooth online multiplayer experience. Now I can play the game with my freinds anywhere in the world, making it much interesting than just a useless uni project.

This project strengthened my skills in object oriented programming in Python, backend architecture, networking, and full project development from concept to a polished, playable product, while also making me comfortable with git and team development.
